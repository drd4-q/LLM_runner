# QNN-LLM-Auto 🚀 — Умный Локальный Ускоритель LLM

**QNN-LLM-Auto** — это высокопроизводительное локальное Android-приложение для ОС **Android 15 (API 35+)**, разработанное для автоматического поиска, инвентаризации и аппаратного ускорения больших языковых моделей (LLM) форматов **GGUF** и **Safetensors** на базе ядер **Hexagon NPU (HTP)** и графических чипов **Adreno GPU** через **Qualcomm AI Engine Direct SDK (QNN)**.

---

## 🌟 Главные Нововедения QNN-LLM-Auto

1. **Автоматический Сканер Системных Blobs:**
   * На старте приложение сканирует системные папки `/vendor/lib64/`, `/vendor/lib/` и `/vendor/gpu/kbc/` на наличие обязательных библиотек Qualcomm.
   * Выводит детальный чек-лист файлов на главном экране.
   * Зелёный статус: **NPU + GPU + CPU АКТИВИРОВАНО** (все файлы найдены).
   * Красный статус: **NPU НЕ НАЙДЕН** (хотя бы один файл отсутствует) — приложение мгновенно и безопасно переходит в софтверный режим GPU/CPU, полностью исключая падения или сбои.

2. **Конвертация Safetensors "На Лету":**
   * Поддержка прямого выбора файлов `.safetensors` из файлового менеджера или кэша HuggingFace.
   * Виртуальное наложение структурного конвертера: приложение эмулирует проход весов через слой динамической квантизации GGUF без создания физических копий на накопителе.

3. **Колоритная Live Terminal консоль:**
   * Логирование в реальном времени всех вызовов NPU, JNI-библиотек, процесса монтирования Root-папок, а также фаллбэк-переключений бэкендов в случае недоступности оборудования.

## 🛠️ Архитектура и Проверяемые Файлы

Для запуска аппаратного тензорного сопроцессора (HTP) сканер проверяет наличие следующих системных объектов:

| Имя файла | Папка поиска | Роль в конвейере инференса |
| :--- | :--- | :--- |
| `com.qti.feature2.ml.so` | `/vendor/lib64/` или `/vendor/lib/` | Связующие биндинги ML API Qualcomm |
| `libQnnSystem.so` | `/vendor/lib64/` или `/vendor/lib/` | Системный менеджер контекстов QNN |
| `libQnnIr.so` | `/vendor/lib64/` или `/vendor/lib/` | Компилятор промежуточных представлений графа |
| `libQnnGpu.so` | `/vendor/lib64/` или `/vendor/lib/` | OpenCL библиотеки для видеоядра Adreno |
| `unified_kbcs_64.bin` | `/vendor/gpu/kbc/` | Скомпилированные шейдерные ядра Adreno |
| `unified_ksqs.bin` | `/vendor/gpu/kbc/` | Пространственные веса оптимизатора матрицы |
| `sequence_manifest.bin` | `/vendor/gpu/kbc/` | Манифест конвейерных вычислений |

## 📦 Сборка и Запуск Проекта

### Требования к окружению:
* **Android Studio Ladybug (2024.2.1+)** или выше.
* **Gradle 8.5+** с Kotlin DSL.
* **Android Core SDK 35+ / SDK 36**.
* **Разрешение Root (Magisk/SU / KernelSU):** Необходимо для прямого монтирования `/vendor/` или работы в режиме Overlay. *При отсутствии Root приложение работает в песочнице локальной симуляции без крашей!*

### Сборка через Gradle:
Выполните сборку отладочного APK в корневой папке проекта:
```bash
gradle assembleDebug
```

Для запуска unit-тестов Robolectric выполните:
```bash
gradle :app:testDebugUnitTest
```

## ⚡ Решение проблем dm-verity и Записи в /vendor

В Android 13+ прямая запись на раздел `/vendor/` заблокирована механизмом **dm-verity** (Read-Only). 

### Senior+ Способы проброса Blobs:

#### Метод А: Magisk-модуль на базе Overlay (Рекомендуется)
Каталог `/vendor` подменяется виртуально без реального изменения разделов:
1. Создайте структуру папок модуля:
   ```bash
   mkdir -p /data/adb/modules/qnn_overlay/system/vendor/lib64
   mkdir -p /data/adb/modules/qnn_overlay/system/vendor/gpu/kbc
   ```
2. Разместите туда полученные blobs и выставьте системные права `0755` и `0644`.
3. Перезагрузите устройство. Приложение мгновенно покажет статус **"HTP + GPU АКТИВИРОВАНО"**.

#### Метод Б: Dynamic Installer (Встроен в интерфейс)
В приложении реализована функция Root-монтажника (кнопка `Копировать Blobs`):
* Приложение пытается подмонтировать раздел `/vendor` в режиме RW с помощью привилегий superuser (`su`).
* В случае неудачи (активная блокировка verity) или детекта Sandbox, приложение автоматически локализует библиотеки во внутреннюю защищённую область:
  `/data/data/com.example/files/qnn_libs/` — сканер распознаёт её как валидный драйверный оверлей.

## 🤖 Запуск Локального Инференса LLM

1. Поместите модель чата (например, `qwen2-1.5b-instruct-q4_k_m.gguf` или модель `.safetensors`) в папку `Download` устройства.
2. Откройте приложение QNN-LLM-Auto.
3. Нажмите кнопку **«Обзор»** для импорта модели через File Picker.
4. Выберите нужный аппаратный профиль:
   * **HTP (NPU):** Рекордная производительность и минимальный расход батареи.
   * **GPU (OpenCL):** Высокая скорость на FP16.
   * **CPU (Kryo):** Референсный фаллбэк-режим.
5. Наберите текст запроса, нажмите **«Запустить инференс»** и наслаждайтесь токенами в реальном времени!
